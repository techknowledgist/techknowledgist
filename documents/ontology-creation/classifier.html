Trainer -- input arguments and invocation

Input to the training (model building) phase is the following:

  1. a file with annotations, with optionally a number of lines to take
  2. a file list that has all the files that the model is trained on
  3. a configuration file with the pipeline configuration
  4. major and minor versions (based on your version and xval variables)

With this I create a bunch of files in the train directory. Let's say I use the following invocation:

% python step4_classify.py --utrain -t data/patents -l en --config pipeline-default.txt --file training-files-v1.txt --annotation-file ../annotation/en/phr_occ.lab --annotation-count 2000 --version standard --xval 1

(many of these options have defaults, but I spelled them all out)

Trainer -- files created

The --train processing step creates the following files in data/t1_train:

utrain-standard.1-info-general.txt

This file contains all setting needed to replicate creating the model, for the above command, the contents of the file are as follows:

version standard xval 1 file_list training-files-v1.txt annotation_file ../annotation/en/phr_occ.lab annotation_count 2000 config_file pipeline-default.txt git commit v0.2-10-g9cbc97c

utrain-standard.1-info-filelist.txt

This contains a local copy of the file list with all training files.

utrain-standard.1-info-config.txt

This contains a local copy of the pipeline configuration. Notice that this is the file used to select a dataset.

utrain-standard.1-info-annotations.txt

Contains a local copy of the annotations file. Only takes the first n lines, where n is the value of the --annotation-count option.

utrain-standard.1-summary-docfeats.txt
utrain-standard.1-phrfeats.txt

Created from the dataset selected with the config file, using just those files in the file list. These two perform the same function as the old --summary setting, gathering all data and putting them in a single file (the difference is that file naming is now more flexible and that the file is added to the train directory rather than the old ws directory.

utrain-standard.1-mallet
utrain-standard.1-MaxEnt.model
utrain-standard.1-MaxEnt.out
utrain-standard.1-MaxEnt.stderr

These are the same as before.
