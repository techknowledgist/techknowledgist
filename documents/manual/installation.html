Ontology Creation - Installation

index > installation

Requirements for running the code:

  1. Python 2.6 or 2.7
  2. Java 1.6
  3. SQLite 3.3.6 or higher (probably any major version of 3 or higher)
  4. xsltproc and xmllint
  5. Git 1.8 (needed only for getting the code, slightly older versions most likely will also work)

The code has been tested on RHEL 6 and Mac OS 10.6.

Installation is a matter of (1) getting the git repository and installing the sub modules, (2) installing third party software (Stanford tools and Mallet classifier), and (3) dowloading the classifier models.

  1. The code is available at the git repository at

    http://www.cs.brandeis.edu/~marc/git/fuse/patent-classifier.git

    When you are reading this document you probably already have the code, but if not, obtain it by 

    $ git clone http://www.cs.brandeis.edu/~marc/git/fuse/patent-classifier.git

    After getting the clone, you need to initialize and update the submodules used by the repository:

    $ git submodule init
$ git submodule update
  2. Stanford Tools.
  3. The code uses version 2.0.7 of the Mallet toolkit1, available at http://mallet.cs.umass.edu/. Other versions will most likely work as well, but were not tested. You can get the version we use at
    http://www.cs.brandeis.edu/~marc/fuse/downloads/tools/
  4. There are small classifer models (for technology classifier and key term extraction) bundled in with the code, which is sufficient when you just try to see if the code runs, but eventually you will want to get a better model. Technology classifier models are available at:
    http://www.cs.brandeis.edu/~marc/fuse/downloads/models/

    There is currently only one model available. You can easily create your own models, as described in the manual section on the technology classifier.

1 Andrew Kachites McCallum. "MALLET: A Machine Learning for Language Toolkit." http://mallet.cs.umass.edu. 2002.
